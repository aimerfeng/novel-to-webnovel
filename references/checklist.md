# 交付前检查清单（阶段 5）

逐项过，全部通过才合并交付。

## 脚本可查项（先跑 qa_check.pl）

- [ ] 残留「」『』 = 0
- [ ] 残留假名注音 = 0
- [ ] 无零字节 / 异常偏小的 chunk 文件
- [ ] chunk 编号连续无缺块

## 结构完整性

- [ ] 章节标题逐一对照原文：一个不多、一个不少、文字未被改动
- [ ] 每章开头结尾与原文对应段落对得上（防止子代理截断或续写）
- [ ] 块接缝处连读通顺（每个接缝抽前后各一段）
- [ ] 原样区间（若有）按用户选择正确处理：原样拼入 / 淡化 / 跳过并注明

## 内容一致性

- [ ] 人名全书统一（grep 每个人名的多种可能写法交叉验证）
- [ ] 称呼体系统一（"XX同学"/直呼名字的规则前后一致）
- [ ] 核心梗/关键道具名称统一（如"项圈""主人点数"）
- [ ] 没有子代理擅自添加的情节、对话、角色

## 风格质量（抽样每章各 2~3 段）

- [ ] 网感词密度 ≤ 千字 3 处；同词全书 ≤ 5 次
- [ ] 严肃/催泪/冲突段落零网感词
- [ ] 吐槽只出现在叙述，未传染给不匹配的角色对话
- [ ] 黑名单词（YYDS/绝绝子等，见 style-guide.md）零出现：
      `grep -c -E "YYDS|绝绝子|栓Q|芭比Q|夺笋|awsl|xswl" 输出.txt`
- [ ] 对话意思与原文一致，没被改写丢失信息

## 交付

- [ ] 输出为新文件（除非用户明确要求覆盖）
- [ ] 向用户报告：总字数、处理章节数、原样区间及处理方式、建议抽读的段落
