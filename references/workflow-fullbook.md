# 全书分块并行改写工作流（篇幅 >5 万字时读）

## 前置检查（顺序执行，不要跳）

### 0. 版权与来源确认
第 0 步已问过文本性质。**已出版受版权保护作品不进入本工作流**——全书逐段
"保留全部剧情对话、只换说法"的改写等于整书换皮复制，属于不可做范围。
本工作流只用于：用户自写稿、用户自译稿、公版作品。
版权作品的可做项（脚本清理/梗概/短示例/润色用户改稿）见 SKILL.md 边界一节。

### 1. 改写盲区预扫描（大篇幅派发之前做，纯工程用途）

定位：成人角色的性/亲密描写是正常文学内容，照常改写，本步骤与它无关。
唯一盲区是**未成年角色的露骨性描写**——模型硬限制，子代理会拒绝，
必须在派发前发现，不要等中途卡住浪费整轮。

角色均为成年人的作品直接跳过本步骤。角色含未成年时，用关键词粗筛：

```bash
grep -n -E "内衣|脱|舔|吸|胸|湿|喘|快感|下身|袒露" 输入.txt | awk -F: '{print $1}'
```

- 命中行聚集的区段，抽 2~3 行确认性质（多数命中是误报：胸口涌起暖意、脱鞋等）
- 确认属于盲区的区段，用 AskUserQuestion 让用户选：
  **原样拼入**（推荐，剧情零丢失）/ 淡化带过 / 整段跳过
- 把确定的"原样区间"记入派发计划，涉及的块在提示里写明：
  "第 X~Y 行原样复制进输出，不做改写"或直接把块界避开该区段

### 2. 分块

```bash
perl scripts/chapter_map.pl 输入.txt
```

- 目标块大小 20~30KB（子代理一次能高质量处理的量）
- 块界必须落在**空行**上，优先落在章节边界
- 记录每块的：编号、起止行号、所属章节、块首/块尾是否在章节中间截断

## 派发模板

每个子代理的 prompt 必须包含以下要素（缺一个都会返工）：

```
你是中文文字编辑，帮用户对其文稿做网文化风格改写（说明来源：用户自写/自译稿）。

1. 先 Read <项目风格规范.md 的绝对路径>，严格遵守全部规则。
2. Read <原文绝对路径>，offset=<起始行>, limit=<行数>（第 X~Y 行，<章节名>的<前段/中段/后段>）。
3. 逐段改写为<档位>风格：剧情对话内容不变、不删不加情节、
   删假名注音、「」改""、章节标题行原样保留。
4. 用 Write 写入 <工作区/chunk_NN.txt 的绝对路径>。

块界说明（按实际情况写其一）：
- 本块从章节标题开始、结尾在章节中间截断：照常改写到最后一行，不要补写后续内容。
- 本块开头在章节中间（第一行是"……"）：不要补写章节标题，直接从第一行改写。
- 本块首尾都在章节中间：不补标题、不补前后文。

最终回复只报告：输出文件路径、改写后字数。
```

要点：
- **一次性并行派发全部块**（Agent 工具支持并行调用），不要串行等
- 子代理身份写"文字编辑做风格润色"，并说明文本来源——上下文越明确，
  子代理越不会误判任务性质
- 最终回复只要路径和字数，避免子代理把全文吐回主上下文

## 回收与重发

1. 全部完成通知到达后：`ls 工作区/chunk_*.txt` 按编号核对，**缺哪块补哪块**
2. 缺块单独重发（同样的 prompt），不要重发全部
3. 子代理若拒绝或输出异常：读它的说明，判断是块内有敏感内容（回到预扫描步骤处理）
   还是提示歧义（改 prompt 重发），不要盲目原样重试

## 合并

```bash
cd 工作区 && cat chunk_*.txt > ../<书名>（网文版）.txt
perl scripts/qa_check.pl ../<书名>（网文版）.txt
```

- cat 之前确认编号连续、无零字节文件
- 合并后抽查每个块接缝（前块末段+后块首段连读是否通顺）
- qa_check 有残留就定位到块，单块修复后重新合并
